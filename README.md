## 💡 My Approach

> *"Don't just solve it — understand it."*

For each problem I try to:

1. **Read carefully** — understand constraints before writing any code
2. **Think first** — sketch an approach on paper or in comments
3. **Code clean** — meaningful variable names, no magic numbers
4. **Revisit** — if a better solution exists, I go back and rewrite it

## 🚀 Getting Started

Want to run any solution locally?

```bash
# Clone the repo
git clone https://github.com/Mazed-Khan/Practicing-Problem-Solving.git

# Navigate to a problem
cd CodeForces/

# Compile & run (C++17)
g++ -std=c++17 -o solution solution.cpp
./solution
```
